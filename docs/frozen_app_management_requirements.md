
# 冻结应用管理需求文档

## 1. 背景与目标

SideLeap 计划新增“冻结应用管理”功能，用于通过 Shizuku 权限通道冻结或解冻指定应用。该功能目标不是复刻完整 Hail，而是在 SideLeap 内提供一个轻量、直接、可批量操作的应用冻结管理页。

第一版以“管理页”为核心，不先接入手势动作体系。用户可以在页面中维护一键操作名单，执行一键冻结或一键解冻，也可以对单个应用直接冻结或解冻。

## 2. 功能范围

本功能包含：

- 新增冻结应用管理页。
- 主页新增“冻结应用管理”入口卡片。
- 使用 Shizuku 执行冻结和解冻。
- 支持用户应用与系统应用显示控制。
- 支持一键冻结选中应用。
- 支持一键解冻选中应用。
- 支持单个应用冻结 / 解冻。
- 支持持久化保存一键名单。
- 支持保护名单。
- 支持保护名单页直接解冻已冻结应用。
- 支持下拉刷新。
- 支持搜索。
- 支持冻结配置纳入备份 / 恢复。

本功能不绑定具体发布版本，先作为独立需求沉淀。

## 3. 权限与执行模型

冻结与解冻使用 Shizuku 权限通道执行。

冻结语义采用 `disable-user`：

```text
冻结：pm disable-user --user 0 <package>
解冻：pm enable <package>
````

普通 Android 权限不能直接冻结其他应用，因此冻结 / 解冻执行能力依赖 Shizuku 可用且已授权。

Shizuku 只用于执行冻结 / 解冻命令。冻结状态查询不依赖 Shizuku，应优先通过 PackageManager 查询当前应用启用状态。

Shizuku 不可用或未授权时：

* 冻结应用管理页仍可进入。
* 应用列表仍可查看。
* 一键名单仍可编辑。
* 保护名单仍可编辑。
* 单个冻结 / 解冻按钮禁用。
* 一键操作 FAB 禁用。
* 不显示 Shizuku 状态提示。
* 不显示 Shizuku 不可用原因图标。
* 不提供打开 Shizuku 的入口。

## 4. 冻结应用管理页

### 4.1 页面入口

主页新增“冻结应用管理”入口卡片，点击进入冻结应用管理页。

### 4.2 页面结构

冻结应用管理页包含：

* 顶部栏

  * 标题：冻结应用管理
  * 右侧盾牌图标：进入保护名单管理页
* 显示系统应用开关
* 顶部信息行

  * `已选：x｜已冻结：y`
* 搜索框
* 应用列表

  * 分组标题：一键名单
  * 分组标题：其他应用
* 右下角主 FAB

  * 主图标：雪花
  * 点击后展开两个雪花图标按钮
  * 一个表示一键冻结
  * 一个表示一键解冻

### 4.3 应用显示范围

冻结应用管理页默认只显示用户安装应用。

页面提供“显示系统应用”开关：

* 关闭时：

  * 只显示用户应用。
  * 搜索只覆盖用户应用。
  * 一键冻结 / 解冻只处理用户应用范围内的一键名单应用。
  * 已选数量只统计用户应用范围内的一键名单应用。
  * 已冻结数量只统计用户应用范围内的冻结应用。

* 开启时：

  * 显示用户应用和系统应用。
  * 搜索覆盖用户应用和系统应用。
  * 一键冻结 / 解冻可处理用户应用和系统应用范围内的一键名单应用。
  * 已选数量统计用户应用和系统应用范围内的一键名单应用。
  * 已冻结数量统计用户应用和系统应用范围内的冻结应用。

系统应用和用户应用同等处理，不设置额外确认、不设置额外禁用规则。

“显示系统应用”开关需要持久保存。

### 4.4 搜索规则

搜索只影响列表显示，不影响一键冻结 / 一键解冻的执行范围。

一键操作目标由以下规则决定：

```text
一键操作目标 = 一键名单 ∩ 当前应用范围 - 保护名单
```

其中“当前应用范围”只受“显示系统应用”开关影响，不受搜索关键词影响。

### 4.5 顶部信息行

冻结应用管理页显示：

```text
已选：x｜已冻结：y
```

`已选：x` 的统计口径：

* 统计当前应用范围内的一键名单应用数量。
* 不受搜索影响。
* 不统计保护名单中的应用。
* 关闭“显示系统应用”时，系统应用即使在一键名单中也不计入。
* 开启“显示系统应用”时，系统应用正常计入。

`已冻结：y` 的统计口径：

* 统计当前应用范围内状态为已冻结的应用数量。
* 不受搜索影响。
* 不统计保护名单中的应用。
* 关闭“显示系统应用”时，只统计用户应用。
* 开启“显示系统应用”时，统计用户应用和系统应用。

### 4.6 列表排序与分组

冻结应用管理页按以下规则显示：

1. 一键名单应用置顶。
2. 非一键名单应用排在后面。
3. 显示分组标题：

   * 一键名单
   * 其他应用
4. 每组内部按应用名排序。
5. 搜索只过滤显示结果，不改变排序和分组规则。
6. 保护名单中的应用不在冻结应用管理页显示。

### 4.7 应用列表项

每个应用列表项包含：

```text
应用图标 + 应用名
包名
雪花图标按钮
一键名单复选框
```

应用图标与应用名区域：

* 显示应用图标。
* 显示应用名。
* 图标颜色表示冻结状态。
* 颜色逻辑沿用快速启动器中冻结状态的现有视觉逻辑。

包名：

* 显示应用包名。

雪花图标按钮：

* 不显示文字。
* 使用雪花图标。
* 颜色表示当前冻结状态。
* 已冻结：高亮色。
* 未冻结：普通 / 弱化色。
* 点击后执行相反动作：

  * 未冻结时点击：冻结。
  * 已冻结时点击：解冻。
* Shizuku 不可用或未授权时禁用。

一键名单复选框：

* 勾选表示加入一键名单。
* 取消勾选表示移出一键名单。
* 已冻结应用仍显示复选框。
* 一键名单语义与当前冻结状态无关：

  * 未冻结应用可加入，用于一键冻结。
  * 已冻结应用可加入，用于一键解冻。

## 5. 右下角一键操作 FAB

### 5.1 主 FAB

冻结应用管理页右下角显示主 FAB：

* 主图标使用雪花。
* 应用列表下滑时自动隐藏。
* 应用列表上滑时显示。
* 当前执行范围内有一键名单应用时启用。
* 当前执行范围内没有一键名单应用时显示但禁用。
* Shizuku 不可用或未授权时禁用。

### 5.2 展开项

点击主 FAB 后展开两个图标按钮：

* 两个按钮都使用雪花图标。
* 不显示文字标签。
* 需要设置无障碍描述。
* 高亮雪花表示一键冻结。
* 弱化雪花表示一键解冻。

批量 FAB 的颜色语义表示“将执行的动作”，不同于单个应用雪花按钮的“当前冻结状态”。

### 5.3 一键冻结 / 解冻行为

一键冻结：

* 目标为当前应用范围内的一键名单应用，排除保护名单。
* 搜索不影响目标范围。
* 当前已冻结的应用跳过。
* 未冻结的目标应用按顺序执行冻结命令。
* 所有命令执行完成后等待 100ms。
* 统一重新查询状态。
* 最终状态为已冻结的应用计入成功数量。
* Toast 显示：

```text
已冻结：x
```

一键解冻：

* 目标为当前应用范围内的一键名单应用，排除保护名单。
* 搜索不影响目标范围。
* 当前未冻结的应用跳过。
* 已冻结的目标应用按顺序执行解冻命令。
* 所有命令执行完成后等待 100ms。
* 统一重新查询状态。
* 最终状态不再是已冻结的应用计入成功数量。
* Toast 显示：

```text
已解冻：x
```

成功数量为 0 时仍按固定格式显示：

```text
已冻结：0
已解冻：0
```

一键操作不显示弹窗，不显示失败详情，不显示跳过数量，不显示命令输出。

## 6. 保护名单

### 6.1 保护名单语义

保护名单用于把应用排除出冻结应用管理范围。

保护名单默认空，不预置任何包名。

被加入保护名单的应用：

* 不在冻结应用管理页显示。
* 不参与冻结应用管理页搜索。
* 不参与一键名单勾选。
* 不参与一键冻结 / 解冻。
* 不计入冻结应用管理页的已选数量。
* 不计入冻结应用管理页的已冻结数量。

加入保护名单时：

* 应用加入保护名单。
* 如果该应用已经在一键名单中，自动从一键名单移除。

移出保护名单时：

* 应用从保护名单移除。
* 回到冻结应用管理页可显示范围。
* 不自动重新加入一键名单。

### 6.2 保护名单入口

冻结应用管理页顶部栏右侧显示盾牌图标。

点击盾牌图标进入保护名单管理页。

## 7. 保护名单管理页

### 7.1 页面功能

保护名单管理页用于维护保护名单。

页面内显示应用列表，支持搜索和勾选加入 / 移出保护名单。

### 7.2 显示系统应用

保护名单管理页有独立的“显示系统应用”开关。

* 默认只显示用户应用。
* 开启后显示用户应用和系统应用。
* 该开关独立于冻结应用管理页的“显示系统应用”开关。
* 该开关需要持久保存。

### 7.3 排序规则

保护名单管理页排序规则：

1. 已加入保护名单的应用置顶。
2. 未加入保护名单的应用排在后面。
3. 两部分内部按应用名排序。
4. 不显示“保护名单 / 其他应用”分组标题。
5. 搜索只过滤显示结果，不改变排序规则。

### 7.4 应用列表项

保护名单管理页应用列表项包含：

```text
应用图标 + 应用名
包名
保护名单复选框
已冻结应用的解冻雪花按钮
```

保护名单复选框：

* 勾选表示加入保护名单。
* 取消勾选表示移出保护名单。
* 加入保护名单时，如果应用已在一键名单中，自动从一键名单移除。

解冻雪花按钮：

* 保护名单页不提供冻结入口。
* 如果应用已冻结，显示高亮雪花按钮，可点击解冻。
* 如果应用未冻结，雪花按钮弱化并禁用。
* 点击解冻后执行解冻命令。
* 执行后等待 100ms。
* 重新查询状态。
* 解冻成功 Toast 显示“已解冻”。
* 状态未变化 Toast 显示“状态未变化”。

## 8. 冻结状态查询

冻结状态查询不依赖 Shizuku。

状态口径：

```text
已冻结：
- PackageManager 返回 DISABLED_USER

未冻结：
- ENABLED
- DEFAULT
- DISABLED
- DISABLED_UNTIL_USED
- 其他非 DISABLED_USER 状态
```

只把 `DISABLED_USER` 视为本功能意义上的“已冻结”。

原因：

* 本功能冻结命令使用 `pm disable-user --user 0`。
* 状态判断应与执行语义一致。
* 不把系统默认禁用、直到使用时启用等其他状态混同为本功能冻结。

## 9. 执行与刷新规则

### 9.1 单个应用冻结 / 解冻

单个应用冻结流程：

1. 执行 Shizuku 冻结命令。
2. 等待 100ms。
3. 重新查询 PackageManager 状态。
4. 如果最终状态为 `DISABLED_USER`，判定为冻结成功。
5. 否则判定为状态未变化。

单个应用解冻流程：

1. 执行 Shizuku 解冻命令。
2. 等待 100ms。
3. 重新查询 PackageManager 状态。
4. 如果最终状态不再是 `DISABLED_USER`，判定为解冻成功。
5. 否则判定为状态未变化。

单个操作反馈：

```text
冻结成功：已冻结
解冻成功：已解冻
失败：状态未变化
Shizuku 不可用：按钮禁用，不通过点击 Toast 提示
```

### 9.2 一键冻结 / 解冻

一键操作执行规则：

1. 读取当前冻结状态。
2. 跳过已符合目标状态的应用。
3. 对剩余目标按顺序执行 Shizuku 命令。
4. 全部命令执行完成后等待 100ms。
5. 统一重新查询 PackageManager 状态。
6. 根据最终状态统计成功数量。
7. Toast 显示成功数量。

成功判定只看重新查询后的最终状态，不以命令返回码为准。

命令输出不展示，也不在结果反馈中显示。

失败项不展示详细原因，一键操作只显示成功数量。

### 9.3 刷新触发点

冻结应用管理页支持下拉刷新。

刷新触发点包括：

* 进入页面时自动加载。
* 用户下拉刷新。
* 单个冻结 / 解冻执行后刷新。
* 一键冻结 / 解冻执行后刷新。
* 保护名单页解冻执行后刷新。

刷新内容包括：

* 应用列表。
* 冻结状态。
* 当前系统应用显示范围。
* 保护名单过滤结果。
* 已选数量。
* 已冻结数量。

## 10. 持久化与备份恢复

### 10.1 持久化内容

需要持久化保存：

* 一键名单。
* 保护名单。
* 冻结应用管理页“显示系统应用”开关。
* 保护名单管理页“显示系统应用”开关。

需求文档不规定具体字段名，不提供数据结构草案，不提供 Kotlin data class。

### 10.2 备份 / 恢复

冻结应用配置需要纳入备份 / 恢复。

恢复备份时：

* 删除当前设备不存在的包名。
* 如果一键名单和保护名单中存在同一个包名，保护名单优先。
* 冲突包名保留在保护名单中，并从一键名单移除。
* 两个“显示系统应用”开关正常恢复。

恢复后不存在的包名不保留。

## 11. 边界情况

### 11.1 Shizuku 不可用

Shizuku 不可用或未授权时：

* 页面可进入。
* 列表可查看。
* 名单可编辑。
* 冻结 / 解冻执行按钮禁用。
* 不显示状态条。
* 不显示原因图标。
* 不提供打开 Shizuku 的入口。
* 不通过点击禁用按钮显示 Toast。

### 11.2 系统应用

系统应用与用户应用同等处理。

* 不设置默认保护名单。
* 不预置桌面、输入法、系统设置、Shizuku、SideLeap 自身等保护项。
* 不做系统应用风险确认。
* 系统应用是否出现在列表中，由对应页面的“显示系统应用”开关决定。

### 11.3 搜索与一键操作范围

搜索只影响可见列表，不影响一键操作范围。

关闭“显示系统应用”时，即使系统应用仍保留在一键名单中，也不参与当前一键冻结 / 解冻。

开启“显示系统应用”后，系统应用才参与当前一键冻结 / 解冻。

### 11.4 保护名单与一键名单冲突

保护名单优先级高于一键名单。

运行时加入保护名单：

* 自动从一键名单移除。

备份恢复时出现冲突：

* 保留在保护名单。
* 从一键名单移除。

### 11.5 已冻结应用进入保护名单

如果应用已经被冻结，然后被加入保护名单：

* 该应用从冻结应用管理页隐藏。
* 该应用不参与一键冻结 / 解冻。
* 用户可以在保护名单管理页看到该应用。
* 用户可以在保护名单管理页直接解冻该应用。
* 用户也可以先移出保护名单，再回冻结应用管理页处理。

## 12. 明确不做项

第一版明确不做以下内容：
* 不做 root 模式。
* 不做 Device Owner 模式。
* 不做 suspend / hide 模式。
* 不做普通权限冻结。
* 不设置默认保护名单。
* 不设置硬性不可冻结名单。
* 不对系统应用做额外保护或二次确认。
* 一键冻结 / 解冻不做二次确认。
* 一键操作不显示结果弹窗。
* 一键操作不显示失败详情。
* 一键操作不显示跳过数量。
* 一键操作不展示 Shizuku 命令输出。
* 不显示 Shizuku 状态条。
* 不显示 Shizuku 不可用原因图标。
* 不提供打开 Shizuku 的入口。
* 暂不绑定具体发布版本。
